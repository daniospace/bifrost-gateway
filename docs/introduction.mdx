AuraLLM is a highly performant, commercial-grade AI Gateway written in Go. It serves as an secure, lightweight bridge connecting your developers' standard OpenAI codebases to any downstream LLM provider (OpenAI, Anthropic, Gemini, DeepSeek, LLaMA, etc.) while enforcing **real-time budget limits, local PII sanitization, and automated model migration audits** on-the-fly.

```python Python (From standard OpenAI Client to AuraLLM in 2 lines!)
import openai

client = openai.OpenAI(
    api_key="gw-team-key-your-secret-here", 
    base_url="https://gateway.yourcompany.com/v1" # Simply swap URL to get strict budget caps!
)
```

## ⚡ Why Mid-Sized Teams Choose AuraLLM

Mid-sized engineering organizations (100 to 1,000 employees) adopt AuraLLM to satisfy budget predictability and private data compliance without the overhead of heavy enterprise security middleware:

*   **Distributed Budget caps:** Atomic budget deduction synchronized globally across your clusters using a distributed **Redis Cache**.
*   **Shadow Routing (Migration Auditor):** Duplicate live production prompts in background goroutines to test cheaper models side-by-side with zero reliability risk.
*   **Local PII Redaction Guard:** Automatically scrub credit cards, SSNs, and secrets on your local servers before prompts ever leave your private network.
*   **OIDC/Okta SSO Integration:** Satisfy procurement security audits out-of-the-box with Single Sign-On and dynamic Role-Based Access Controls.

## 🛠️ Performance & Overhead

Because AuraLLM is built in Go, it features near-zero overhead under high-concurrency mid-market workloads:

*   **RAM Footprint:** ~15MB at idle.
*   **Startup Latency:** less than 10ms.
*   **Edge deployment:** Runs as a lightweight, zero-dependency 20MB Docker container or Kubernetes sidecar.
